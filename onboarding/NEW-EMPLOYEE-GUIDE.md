## Day 1 上午：連上你的工作桌面

### 什麼是 VDI？

VDI 就像「雲端電腦」——你不需要在自己的電腦上安裝任何東西，所有開發工具都裝在雲端的一台虛擬電腦裡。你只要透過「遠端桌面」連上去，就像坐在那台電腦前面一樣操作。

### 連線步驟

**Step 1：打開「遠端桌面」應用程式**

在你的 Windows 電腦上，搜尋並打開 **Remote Desktop** 或 **遠端桌面**：

- 按下鍵盤的 `Windows 鍵`（左下角那個旗子圖案）
- 輸入 `Remote Desktop`
- 點開 **Remote Desktop** 應用程式

> [!TIP]
> 如果找不到，也可以到 Microsoft Store 搜尋 **Azure Virtual Desktop** 下載安裝。

**Step 2：登入**

使用 MIS 給你的帳號密碼登入：

| 欄位 | 填什麼 |
|------|--------|
| 使用者名稱 | MIS 給你的 AD 帳號（類似 `tony.lin@company.com`） |
| 密碼 | MIS 給你的初始密碼 |

> [!IMPORTANT]
> 第一次登入後，系統會要求你**修改密碼**。請設一個你記得住的強密碼（8 位以上，包含大小寫英文和數字）。

**Step 3：看到桌面 = 成功！**

登入後你會看到一個 Windows 11 桌面，上面已經有幾個捷徑：

| 桌面圖示 | 它是什麼 | 比喻 |
|----------|----------|------|
| 📝 **VS Code** | 程式碼編輯器 | 你的「Word」，但是寫程式用的 |
| 🌐 **XAMPP Control Panel** | 本地伺服器控制台 | 你的「小型伺服器開關」 |
| 🔗 **WordPress 測試站** | 本地測試用的 WordPress 網站 | 你的「練習場」，怎麼搞都不怕壞 |
| 📂 **外掛開發目錄** | 放置外掛程式碼的資料夾 | 你的「工作間」 |

## Day 1 下午：認識你的開發工具

### 工具 1：XAMPP — 你的小型伺服器

WordPress 是一個「網站」，要跑起來需要三個東西：網頁伺服器（Apache）、資料庫（MySQL）、還有 PHP。XAMPP 就是把這三個東西打包在一起的工具箱。

**啟動 XAMPP：**

1. 雙擊桌面的 **XAMPP Control Panel**
2. 點 Apache 旁邊的 **Start** 按鈕 → 看到綠色表示啟動成功
3. 點 MySQL 旁邊的 **Start** 按鈕 → 看到綠色表示啟動成功

```
┌────────────────────────────────────────────┐
│          XAMPP Control Panel               │
├──────────┬──────────┬──────────────────────┤
│ Module   │ Status   │ Actions              │
├──────────┼──────────┼──────────────────────┤
│ Apache   │ 🟢 綠色  │ [Stop]               │  ← 這樣就對了
│ MySQL    │ 🟢 綠色  │ [Stop]               │  ← 這樣就對了
│ FileZilla│          │ [Start]              │  ← 不用管它
│ Mercury  │          │ [Start]              │  ← 不用管它
│ Tomcat   │          │ [Start]              │  ← 不用管它
└──────────┴──────────┴──────────────────────┘
```

> [!CAUTION]
> 只需要啟動 **Apache** 和 **MySQL** 兩個就好！其他的不要去碰。

### 工具 2：WordPress 測試站 — 你的練習場

XAMPP 啟動後，打開瀏覽器（Edge 或 Chrome），在網址列輸入：

```
http://localhost/my-tiny-wpdev/wp-admin/
```

用以下帳號登入：

| 欄位 | 填什麼 |
|------|--------|
| 使用者名稱 | `my_tiny_wpdev` |
| 密碼 | 請找貓哥取得 🐱 |

登入後你就會看到 WordPress 的管理後台，這就是我們開發外掛的「遊樂場」。

### 工具 3：VS Code — 你的程式碼編輯器

1. 雙擊桌面的 **VS Code** 圖示
2. 你會看到一個深色的編輯器介面
3. VS Code 已經幫你裝好了 4 個擴充套件（Extensions），讓你寫程式更方便：

| 擴充套件 | 做什麼用 | 比喻 |
|----------|----------|------|
| **PHP Intelephense** | PHP 程式碼的自動完成和錯誤檢查 | 像手機輸入法的「聯想字」 |
| **WordPress Hooks IntelliSense** | WordPress 專用函式的自動提示 | 像 WordPress 的「字典」 |
| **Tailwind CSS IntelliSense** | CSS 樣式的自動完成 | 像設計用的「色票本」 |
| **GitLens** | 看程式碼的修改歷史 | 像 Word 的「追蹤修訂」 |

### 工具 4：Git — 程式碼的「時光機」

Git 是版本控制工具——想像你在寫文章，每次按「存檔」就會建立一個版本，之後隨時可以回到任何一個版本。Git 就是做這件事的，但是更強大。

你暫時不需要深入了解 Git，只要知道：
- 所有程式碼都存在 **GitHub**（雲端程式碼倉庫）上
- 你的工作會透過 Git「推」到 GitHub 上，跟團隊共享

### 工具 5：Antigravity — 你的 AI 助手

Antigravity 是 Google 的 AI 編程助手，已經幫你裝好了。它可以：
- 幫你寫程式碼
- 解釋程式碼是什麼意思
- 按照團隊的標準流程（SKILL.md）自動幫你建立外掛

> 你現在看到的這份文件，就是用 Antigravity 寫出來的 😉

## ❓ 常見問題 FAQ

### Q：XAMPP 的 Apache 啟動失敗，顯示紅色？

**A：** 最常見的原因是 port 80 被其他程式佔用了（例如 Skype 或 IIS）。

解決方法：找貓哥協助 🐱

### Q：WordPress 測試站打開是空白頁面？

**A：** 確認 XAMPP 的 Apache 和 MySQL 都是綠燈。如果是，可能是設定問題。

解決方法：找貓哥協助 🐱

### Q：GitHub 登不進去？

**A：** 確認你使用的是公司信箱帳號登入，選擇 SSO（Single Sign-On）方式。如果還是不行，可能是 MIS 還沒把你加入 Organization。

解決方法：找貓哥協助 🐱

### Q：VS Code 打開後沒有看到擴充套件？

**A：** 點左邊側邊欄的方塊圖示（Extensions），搜尋 `PHP Intelephense` 看看是不是已安裝。如果沒有，通知 MIS 補裝。

### Q：我不小心把什麼東西弄壞了怎麼辦？

**A：** 別緊張！這就是測試環境的意義——就是讓你放心地搞壞東西的地方。找貓哥，一起看看怎麼修 🐱
